# 设计规范交付包 — 使用说明

> 本文档是给**产品经理**看的操作指南，不需要懂代码，按步骤操作即可。

---

## 第一步：把这个文件夹用 Cursor 打开

用 Cursor 打开整个 `交付包` 文件夹（File → Open Folder），**不要只打开单个文件**。

Cursor 打开文件夹后，会自动读取 `.cursorrules` 文件，AI 就已经知道了所有设计规范和使用规则，不需要你额外说明。

---

## 第二步：让 AI 初始化项目（只需做一次）

在 Cursor 的对话框里，直接说：

> **"帮我初始化项目"**

AI 会自动完成：
- 创建 Vue 3 项目
- 安装 Element Plus 依赖
- 配置主题文件
- 注册所有封装组件

---

## 第三步：描述你要做的页面

初始化完成后，直接告诉 AI 你要做什么就行了，不需要任何特殊关键词。

**举例说明：**

---

### 想做一个账号列表页

> 做一个账号列表页，顶部有 4 个数据统计卡片（总账号数、活跃账号、今日新增、本月新增），然后是筛选区（账号名称、所属分组、使用状态、创建时间），下面是表格（列：账号名称、分组、状态、创建时间、操作），表格右上角有新增按钮，点新增弹出表单弹窗。

---

### 想做一个任务管理页

> 做一个任务管理页，面包屑显示"任务管理 / 任务列表"，筛选条件有任务名称、负责人、任务状态、截止日期，表格显示任务名称、负责人、优先级、状态、截止日期、操作列（查看、编辑、删除）。

---

### 想调整某个页面

> 把刚才那个页面的表格加上斑马纹，操作列的删除按钮改成红色危险按钮。

---

## 文件夹内容说明

```
设计规范项目/（整个文件夹就是交付物）
│
├── .cursorrules                           ← AI 自动读取的规则（无需操作）
├── README.md                              ← 本文件
│
├── colors.md / typography.md              ← 基础规范：色彩、字体
├── spacing.md / border.md                 ← 基础规范：间距、边框
│
├── components/                            ← 26 个组件设计规范
├── instances/                             ← 3 个封装组件用法说明
│
├── public/
│   └── el-plus-theme.css                  ← 设计主题样式文件
│
└── .vitepress/theme/components/
    ├── AppSidebar.vue                     ← 左侧导航组件源码
    ├── AppTopbar.vue                      ← 顶部栏组件源码
    ├── AppFilter.vue                      ← 表格筛选组件源码
    └── AppStatCard.vue                    ← 数据统计卡片组件源码
```

> 设计规范更新后，重新压缩发送整个项目文件夹即可，无需单独整理文件。

## 常见问题

**Q：页面效果和设计稿不一样怎么办？**
> 告诉 AI："对照设计规范检查一下颜色和间距，确保使用的是 el-plus-theme.css 里的变量。"

**Q：想要的布局 AI 没做对怎么办？**
> 更具体地描述，比如："左侧是固定的导航栏，右侧是内容区，内容区顶部有面包屑导航。"

**Q：AI 用了我不认识的组件怎么办？**
> 告诉 AI："所有 UI 组件只能用 Element Plus，不要用其他库。"

---

## 注意事项

1. **每次都在同一个 Cursor 项目里工作**，不要新建项目，否则规则文件会丢失
2. **描述越具体越好**，包括：页面名称、有哪些数据、表格有哪些列、有哪些操作
3. **不需要说"用 Element Plus"、"按设计规范"**这类话，`.cursorrules` 已经告诉 AI 了
